# ПитайБаце

„ПитайБаце" е уеб приложение (тикет система), в което регистриран клиент
заплаща услуги за своя уебсайт и създава тикети, а експерт („пустиняк")
обработва заявката до нейното решаване. Еднократната услуга дава право на
един тикет, а абонаментът — на тикети без ограничение. Комуникацията се
води чрез съобщения, а тайните за сайто (достъпи до сайта и хостинга) се
съхраняват криптирани.

## Технологии

- Python 3.13, Django 5.2
- SQLite за разработка
- `cryptography` (Fernet) за криптиране на тайните

## Структура на приложенията

- **accounts** — собствен потребителски модел с роли клиент/експерт,
  регистрация, вход, профил със снимка и клиентско табло.
- **departments** — отдели, по които се разпределят експертите и тикетите.
- **services** — каталог с услуги, групирани по отдели.
- **orders** — поръчки и симулирано плащане зад `PaymentGateway`
  интерфейс.
- **tickets** — тикети със съобщения, прикачени файлове, приоритети,
  крайни срокове и криптирани тайни за сайто.
- **logs** — одитен лог на действията в системата.
- **staff** — Баце панел за експерти и администратори.

## Настройка за разработка

```bash
python -m venv venv
source venv/bin/activate
pip install -r requirements.txt
cp .env.example .env
python manage.py migrate
python manage.py createsuperuser
python manage.py runserver
```

В `.env` задайте `SECRET_KEY`, `DEBUG=True` и `FERNET_KEY` — ключът за
криптиране се генерира с:

```bash
python -c "from cryptography.fernet import Fernet; print(Fernet.generate_key().decode())"
```

## Преглед на архитектурата

Потребителският модел е собствен, с роля клиент или експерт; експертът
задължително принадлежи към отдел, а администраторът е стандартен Django
суперпотребител. Всяка промяна на статус на тикет минава през една точка
(`tickets/services.py`), която валидира прехода, записва го в одитния лог
и уведомява клиента по имейл.

Контролът на достъпа е строг: клиентът вижда само своите поръчки и
тикети, експертът — само тикетите на своя отдел, а Баце панелът е
достъпен единствено за експерти и администратори. Вътрешните бележки на
експертите никога не се показват на клиента. Затворен тикет не приема
нови съобщения — само експерт може да го отвори отново.

Тайните за сайто се пазят криптирани с Fernet и в базата никога няма чист
текст. Разкриват се само на назначения по тикета експерт, само след
изрично действие („Виж тайните"), и всяко разкриване се записва в лога.

Плащането е симулирано зад `PaymentGateway` интерфейс с валидация на
данните за карта, без да се съхраняват — реални интеграции (карта /
PayPal / ePay) могат да се добавят по-късно, без промяна в останалия код.
Имейлите ползват стандартния Django backend — конзолен при разработка,
SMTP в останалите случаи. Прикачените файлове се валидират по размер и
разширение.
